## 1. 文档目的与范围

本文档定义 MateLink App 的 **UI 视觉规范、信息架构、页面级交互需求与跳转逻辑**，作为前端实现的直接依据。功能需求、技术架构、数据模型见现有 `glm/` 与 `mimo/` PRD，本文档与之互补，专注于"页面长什么样、点哪里去哪里、显示什么数据"。

## 2. 产品概述

### 2.1 产品定位

把 TeslaMate 的数据装进口袋 — 为已自托管 TeslaMate 的 Tesla 车主提供原生移动端体验，替代浏览器访问 Grafana。

### 2.2 核心价值

| 价值点 | 说明 |
|--------|------|
| 数据主权 | 数据存用户自己服务器，不经过第三方云 |
| 深度分析 | 充放电效率、电池衰减、驾驶行为等官方 App 不提供的数据 |
| 移动便捷 | 随时随地查看车辆状态，不再依赖电脑 |
| 实时通知 | 充电完成、异常状态等推送提醒 |

### 2.3 目标用户

- 已部署 TeslaMate 的 Tesla 车主（技术爱好者、数据驱动型）
- 有 NAS/服务器、愿意部署 Docker 的 Tesla 车主

## 3. 设计系统规范

提供两套主题，用户可在「设置 > 主题」切换，默认跟随系统。

### 3.1 主题 A：High-Tech Velocity（深色玻璃拟态·车载座舱）

> 适合夜间/车载场景，沉浸式仪表盘体验。

| 要素 | 规范 |
|------|------|
| 背景 | 深黑蓝 `#0A0E14`（Tier1）/ `#10141A`（Tier2 卡片）|
| 主色 | 电光蓝 `#00A3FF`（操作/充电状态/主按钮）|
| 辅色 | Tesla 红 `#E01E35`（品牌/告警）/ 在线绿 `#4AE183`（在线状态/成功）|
| 字体 | Inter（正文/标题）+ JetBrains Mono（数字/标签，tabular-nums）|
| 圆角 | 卡片 16px / 按钮 8px / Chip 全圆角 |
| 层级 | Glassmorphism：backdrop-blur(20px) + 5% 白填充 + 1px rgba(255,255,255,0.1) 边框 |
| 发光 | 圆形仪表 8px stroke + `0 0 15px #00A3FF` 外发光 |

### 3.2 主题 B：Precision Minimalist（白色简约·瑞士风）

> 适合日间/专业数据场景，克制理性。

| 要素 | 规范 |
|------|------|
| 背景 | 纯白 `#FFFFFF`（对比度 16:1, WCAG AAA）|
| 主色 | 黑 `#171717`（操作/标题）|
| 点缀 | 金 `#A16207`（仅关键 CTA/品牌徽章，克制使用）|
| 状态色 | 在线绿 `#059669` / 充电橙 `#F59E0B` / 警告红 `#DC2626`（仅语义）|
| 字体 | Inter（正文）+ JetBrains Mono（数字，tabular-nums）|
| 圆角 | 卡片 8px / 按钮 4px / Chip 全圆角 |
| 层级 | Flat Layering：1px `#E5E5E5` 细边框，**无阴影**，靠边框+留白分层 |
| 间距 | 8px 基准，慷慨留白 24-32px |

### 3.3 通用规范

- **数字一律 JetBrains Mono + `font-variant-numeric: tabular-nums`** 右对齐，防止数据刷新时布局抖动
- **底部 4-Tab 导航栏**：仪表盘 / 行程 / 充电 / 更多（当前页高亮）
- **图标**：1.5px 线性图标，Material Symbols Outlined
- **图表**：1px 细线，纯色块无渐变（瑞士风）/ 带发光晕（深色风）
- **按压态**：`active:scale-0.98` + 200ms `cubic-bezier(0.16,1,0.3,1)`

## 4. 信息架构与导航流


### 4.3 关键断链修复记录

> 以下断链在原型评审中发现并修复，前端实现时必须保留：

| 断点 | 修复方式 |
|------|----------|
| L1 列表 → L2 详情卡片无点击事件 | 列表卡片 `cursor-pointer` + `onClick` 跳转 + `active:scale-98` 反馈 |
| 充电历史缺实时态入口 | 置顶「正在充电」实时卡（金色左竖线 + 绿色脉动点），点击进当前充电 |
| 月度摘要条是死数据 | 摘要条可点击 → 钻取成本分析 |
| 筛选图标无实现 | 筛选维度：类型(AC/DC/全部) × 时间(周/月/年) × 地点 |

## 5. 页面详细需求

### 5.1 L1 仪表盘 (Dashboard)

**目的**：一屏掌握车辆实时状态。

**布局**：
1. 顶栏：左「Tesla Model 3」标题 + 右绿色脉动点 + 「在线」Chip
2. 核心卡：左大圆环电池进度（78% 中心数字）+ 右大数字「312 km」预估续航
3. 2×2 信息网格：上次行程 31.2 km / 上次充电 42 kWh / 车内温度 24°C / 车门已锁
4. 车辆状态栏：已锁 · 哨兵关闭 · 空调关闭 · 已连接（1.5px 线性图标）
5. 胎压卡：FL/FR/RL/RR 2.5/2.5/2.4/2.4 bar
6. 7 天电池趋势柱状图（今天高亮）
7. 温度卡：车内 24° / 车外 28°

**数据字段**：`battery_level`, `est_battery_range_km`, `last_drive_distance_km`, `last_charge_kwh`, `inside_temp_c`, `outside_temp_c`, `locked`, `sentry_mode`, `is_climate_on`, `plugged_in`, `tire_pressure_fl/fr/rl/rr`

**交互**：电池环点击 → 电池健康；上次行程卡点击 → 行程详情

---

### 5.2 L1 行程历史 (Drives)

**目的**：浏览历史行程，快速进入详情。

**布局**：
1. 顶栏：标题「行程历史」+ 日历图标（按月切换，显示「2025年7月」）+ 筛选图标
2. Bento 2 列统计：总里程 1,248 KM / 平均效率 148 Wh/km（绿色）
3. 行程卡片列表（每卡 cursor-pointer → 行程详情）：
   - 顶行：📍出发地 → 虚线 → 🏁到达地 + 分类 Chip（通勤/旅行/出差）
   - 中部 3 列：距离 / 时长 / 日期
   - 底行（浅灰底）：最高速度 + 电量变化 + `chevron_right`
   - 高效行程（<155 Wh/km）右侧绿色「高效」徽章带发光
4. 底部：「显示最近 30 天的数据」灰小字

**数据字段**：`start_address`, `end_address`, `distance_km`, `duration_min`, `start_date`, `avg_speed_kmh`, `max_speed_kmh`, `consumption_kwh`, `efficiency_wh_per_km`, `start_battery_pct`, `end_battery_pct`, `category`(auto-tagged)

**交互**：月份切换刷新；卡片点击 → 行程详情；默认显示最近 30 天

---

### 5.3 L1 充电历史 (Charges)

**目的**：浏览充电记录，监控实时充电。

**布局**：
1. 顶栏：标题「充电历史」+ 日历 + 筛选（AC/DC/全部 × 时间）
2. Bento 2 列统计：总电量 284 kWh（金色）/ 总费用 ¥186.50
3. 【置顶·仅充电中显示】正在充电实时卡：金色左竖线 + 绿色脉动点 + 「正在充电」Chip + 78% 大数字 + 实时功率 7.4 kW + 预计完成 21:45 + `chevron_right` → 当前充电
4. 历史卡片列表（每卡 cursor-pointer → 充电详情）：
   - 顶行：充电地点 + 地址 + 大数字「+38.5」金色 + kWh
   - 中部：电量变化横向条（65%→100%，金色填充段）+ 时长
   - 底行（浅灰底）：日期 + 类型 Chip（AC 蓝边框 / DC 橙边框）+ 功率 + 费用 + `chevron_right`
   - DC 快充项左侧橙色 `#F59E0B` 竖线标记
5. 月度摘要条可点击 → 成本分析

**数据字段**：`location_name`, `address`, `date`, `charge_kwh`, `start_battery_pct`, `end_battery_pct`, `duration_min`, `charge_type`(AC/DC), `power_kw`, `cost_cny`, `is_charging_now`

**交互**：实时卡仅充电中置顶显示；摘要条点击钻取成本分析

## 6. 数据来源与 API

### 6.1 数据流

```
Tesla 车辆 → TeslaMate(Elixir) → PostgreSQL + MQTT → TeslaMateApi(Go REST) → MateLink App
```

### 6.2 关键 API 端点（TeslaMateApi）

| 端点 | 用途 | 对应页面 |
|------|------|----------|
| `GET /api/v1/cars` | 车辆列表 | 仪表盘、所有页 |
| `GET /api/v1/cars/{id}/status` | 实时状态 | 仪表盘、当前充电 |
| `GET /api/v1/cars/{id}/drives` | 行程列表 | 行程历史 |
| `GET /api/v1/cars/{id}/drives/{drive_id}` | 行程详情 | 行程详情 |
| `GET /api/v1/cars/{id}/charges` | 充电列表 | 充电历史 |
| `GET /api/v1/cars/{id}/charges/{charge_id}` | 充电详情 | 充电详情 |
| `GET /api/v1/cars/{id}/battery-health` | 电池健康 | 电池健康 |
| `GET /api/v1/cars/{id}/updates` | 固件更新 | 固件版本 |
| 统计聚合（自实现） | 月度/年度/成本/效率 | 统计、成本、效率等 |

### 6.3 降级策略

- **Network-First + 缓存降级 + Mock 可切换**（`DelegatingCarRepository`）
- 无高频数据时，行程详情曲线显示「模拟数据 — 基于行程摘要」
- 离线时显示上次缓存数据 + 灰色「离线」标签

### 6.4 中国本地化

- 高德地图 + GCJ-02 坐标转换（原始 WGS-84），HK/Macau 旁路
- 分时电价 TOU：成本统计按时段电价
- 金额 ¥，距离 km，温度 °C
