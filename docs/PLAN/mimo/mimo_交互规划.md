# MateLink Web 交互规划

> 品牌：MateLink
> 技术：React + Vite + Tailwind + Recharts + Leaflet
> 日期：2026-06-23
> 目的：Web 原型确认交互 → 再写原生 App

---

## 全局交互

### 导航
- 左侧边栏：Dashboard / Drives / Charges / Battery / Statistics / Heatmap / Top Destinations / Efficiency / Settings
- 移动端：底部 Tab Bar（Dashboard / Drives / Charges / More）
- 点击页面标题返回顶部

### 主题
- 亮色/暗色切换（右上角开关）
- 跟随系统自动切换
- 车色强调色：根据当前车辆 exterior_color 动态生成主题色

### 离线状态
- 顶部黄色 banner："Offline - showing last known status"
- 数据来源：localStorage 缓存

### Mock 模式
- 设置页开关："Enable Mock Mode"
- 开启后顶部紫色 banner："Mock Mode - All data is fake"
- 内置 2 辆虚拟车

## 1. Dashboard

### 展示内容

| 区域 | 内容 | 数据源 |
|------|------|--------|
| 顶部栏 | 车辆名称 + 状态徽章 + 设置图标 | /status |
| 车辆图 | 2D 车辆图（按 exterior_color 匹配） | /cars |
| 电量区 | 电量百分比（大数字）+ 续航里程 | /status |
| 信息网格 | 6 个卡片：位置+海拔、里程、车内/外温度、胎压、车门锁、充电线 | /status |
| 状态行 | 空调状态、充电上限%、高电量警告 | /status |
| 充电中 | 额外卡片：充电功率、已充电量、预计剩余 | /status |
| 趋势图 | 7 天电量趋势折线图 | /status 历史 |

### 交互

| 交互 | 行为 | 细节 |
|------|------|------|
| **自动刷新** | 每 5s 请求 /status | Loading 时不显示骨架屏，静默更新 |
| **下拉刷新** | 手动触发立即拉取 | 顶部 spinner 动画 |
| **点击车辆名** | 弹出车辆切换 Modal | 列表显示所有车辆，点击切换 |
| **点击状态徽章** | 高亮 + 显示状态详情 tooltip | online:绿 / driving:蓝 / charging:橙 / asleep:灰 |
| **点击信息卡片** | 跳转对应详情页 | 位置→地图、胎压→胎压详情 |
| **趋势图 hover** | 显示日期+电量 tooltip | 垂直参考线 + 数据点高亮 |
| **趋势图缩放** | 鼠标滚轮/双指缩放 | X 轴时间范围变化 |
| **充电中卡片** | 实时更新功率和电量 | 5s 刷新，进度条动画 |
| **车门锁图标** | 🔒/🔓 显示锁状态 | 点击显示 "Locked since X" |
| **充电线图标** | ⚡ 显示插枪状态 | plugged_in:绿 / unplugged:灰 |
| **空调图标** | ❄️ 显示空调状态 | active:蓝 / inactive:灰 |
| **充电上限** | 显示 "Limit: 85%" | 超过 90% 显示黄色警告 |
| **海拔** | 位置卡片下方显示海拔 | 格式: "海拔 120m" |

### 状态徽章颜色

```
online:    bg=#E8F5E9 text=#2E7D32  (绿)
driving:   bg=#E3F2FD text=#1565C0  (蓝)
charging:  bg=#FFF3E0 text=#E65100  (橙)
asleep:    bg=#F5F5F5 text=#616161  (灰)
offline:   bg=#EEEEEE text=#424242  (深灰)
```

## 12. 首次配置

| 步骤 | 内容 |

| 交互 | 行为 |
|------|------|

| **失败动画** | 红色叉 + 错误分类：URL不可达 / 401 / 超时 / TLS |
| **进入 Dashboard** | 点击按钮 → 淡入 Dashboard 页面 |
| **跳过** | 可选："Use Mock Data" 按钮直接进入 |
